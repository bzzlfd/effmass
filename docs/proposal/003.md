# [proposal-003] 双向球贝塞尔递推器（Bidirectional Spherical-Bessel Iterator）

## 背景

当前 `SphericalBesselJ` 是一个**单向**递推器：内部保存 `j_l` 和 `j_{l+1}`，只支持 `++`（前进到 `l+1`）。

在某些算法中（例如自适应截断、试探性增加角动量后再回退），可能需要双向移动。

## 方案

将内部状态从两级扩展为三级：`j_{l-1}`、`j_l`、`j_{l+1}`，并把"当前指针"固定在中间的 `j_l`。

递推公式本身是对称的：

```
j_{l+1} = (2l+1)/x * j_l - j_{l-1}
j_{l-1} = (2l+1)/x * j_l - j_{l+1}
```

因此保存 `j_{l-1}` 和 `j_{l+1}` 足以反推任意方向。

### 状态设计

```cpp
class SphericalBesselJBidirectional {
    double x_;       // |x|
    int l_;          // current l
    double jm1_;     // j_{l-1}
    double j0_;      // j_l  (current value)
    double jp1_;     // j_{l+1}
    bool neg_;
};
```

### 操作

| 操作 | 计算 | 新状态 |
|------|------|--------|
| `++` (到 `l+1`) | `j_{l+2} = (2l+3)/x * j_{l+1} - j_l` | `j_l, j_{l+1}, j_{l+2}` |
| `--` (到 `l-1`) | `j_{l-2} = (2l-1)/x * j_{l-1} - j_l` | `j_{l-2}, j_{l-1}, j_l` |

每次移动只需**一次递推公式计算**，O(1)。

### 边界处理

- 初始化时需要同时准备 `j_{-1}`、`j_0`、`j_1`（其中 `j_{-1} = cos(x)/x`）
- `--` 到 `l < -1` 需要禁止或抛异常
- `x=0` 时所有 `j_l (l>0)` 为 0，需短路处理避免除零

## 数值可逆性分析

`++` 与 `--` 的递推公式互为代数逆：

```
++ : j_{l+2} = A·j_{l+1} - j_l        (A = (2l+3)/x)
-- : j_{l-1} = A'·j_l - j_{l+1}       (A'= (2l+1)/x)
-- 等价于: j_{l-1} = (2l+1)/x·j_l - j_{l+1}
```

### 单步可逆：`++` 然后 `--`

`++` 计算 `j_{l+2} = A·j_{l+1} - j_l`，状态移为 `(j_l, j_{l+1}, j_{l+2})`。`j_l` 和 `j_{l+1}` **未被修改**，只被移位。

`--` 恢复 `j_{l-1}` 时计算 `(2l+1)/x·j_l - j_{l+1}`，使用的 `j_l` 和 `j_{l+1}` 与 `++` 前完全相同。因此在 IEEE 754 下，**bit-identical 可逆**。

### 多步可逆：`++ ++ ++ -- -- --`

不再是 bit-identical，但误差有界且不积累。

``++`` 的核心是 catastrophic cancellation：`A·j_{l+1} ≈ j_l`，结果 `j_{l+2}` 远小于两个操作数。但浮点减法的性质是**绝对误差正比于结果**而非输入：

```
j_{l+2}' = (A·j_{l+1} - j_l)(1 + δ)     |δ| ≤ ε/2
```

恢复时：

```
j_l' = A·j_{l+1} - j_{l+2}'
     = A·j_{l+1} - (A·j_{l+1} - j_l)(1 + δ)
     = j_l + j_{l+2}·δ
```

恢复误差 `|j_l' - j_l| = |j_{l+2}|·|δ| ≤ |j_{l+2}|·ε/2`。

由于 `j_{l+2} ≪ j_l`（当 `l ≫ x`），相对误差 `|j_l' - j_l| / |j_l| ≪ ε/2` —— 远比机器精度低。

同理，`N` 步前进再 `N` 步后退，每步的 cancellation 误差只正比于该步产生的小值，**误差线性累积**（`O(N·ε·j_{l+N})`），不会指数增长。各步间没有交叉污染。

### 结论

不会产生"不可逆"级别的漂移：小步长随机游走后仍可以高精度回到原点。

## 优势

1. **双向 O(1)**：`++`/`--` 代价相同
2. **无额外堆分配**：三个 `double` 的固定开销
3. **数值可逆**：在同一 `l` 附近交错 `++`/`--` 不会产生漂移（因为两边都是直接递推，不是"撤销"）

## 劣势

1. **状态复杂度上升**：从 2 个 `double` 变成 3 个，初始化需要 `j_{-1}`
2. **心智负担**：使用者需要理解三级滑动窗口的语义
3. **使用场景有限**：在赝势/电子结构计算中，通常只需顺序 `0...l_max`，极少需要回退

## 结论

暂不单点实现。若未来出现"试探-回退"类算法需求，再引入此双向递推器。
