# [proposal-006] H|ψ⟩ 非定域节循环次序与缓存策略分析

## 背景

在 `Hamiltonian::Callable::operator()` 的非定域赝势节中，需要计算三类数据：

| 数据 | 符号 | 依赖 τ？ | 计算成本（每 G-vector） |
|------|------|----------|----------------------|
| β(q) 表插值 | β | ❌ 否 | ~15 cycles（表查 + 三次插值） |
| 实球谐函数 Y_lm | Y | ❌ 否 | 0 cycles（`get()` 返回 const ref） |
| 结构因子 S(g,k,τ) | S | ✅ 是 | ~35 cycles（复指数 + 3 复乘） |

然后做两遍 ng_ 循环：

```
inner  pass:  inner = Σ_g  conj(β·Y·S) · ψ[g]     —— 读 β, Y, S, ψ
update pass:  hpsi[g] += λ · inner · β·Y·S         —— 读 β, Y, S, 写 hpsi
```

任何方案下这两遍的总工作量相同（n_atom × n_proj × n_q），不在比较范围内。

关键观察：β 和 Y 不依赖 τ，S 依赖 τ。不同的循环次序 + 缓存策略决定了谁被重复计算。

## 符号约定

| 符号 | 含义 | 典型值 |
|------|------|--------|
| n_atom | 同种元素的原子数 | 10–100 |
| n_β | (l, ib) 对数（l_max=3, nb_l=2） | 8 |
| n_proj | 总投影子通道数 Σ (2l+1)×nb_l | 32 |
| n_q | G-vector 数量 | ~1000 |
| C_β | β 插值每 G-vector 成本 | ~15 cycles |
| C_sf | 结构因子每 G-vector 成本 | ~35 cycles |

## 四种方案

### 方案 A：当前循环次序 + 缓存 β

```
β 提前到 for type 层计算，存到 n_β 个 vector 中。原子循环内 sf_vals 按需算。
```

```cpp
for (auto&& type : atom.eachType()) {
    // 预计算 β(q) — 不依赖 τ，1 次
    for (auto&& bi : blocks)
        for (int ib = 0; ib < bi.nb; ++ib)
            beta_cache[bi.l][ib] = interpolate(...);

    for (auto&& ad : atom.eachTypeAtom(type.ityp)) {
        sf_vals[ig] = sf(g, k);               // 1× per atom
        for (auto&& bi : blocks)
            for (int ib = 0; ib < bi.nb; ++ib)
                for (int m = -bi.l; m <= bi.l; ++m)
                    inner / update pass        // β, Y 从 cache/ref 读
    }
}
```

### 方案 B：atom 在最内层 + 缓存 sf

```
sf_vals 提前到 for type 层对所有原子算好存好。for type 内重排为
for bi → for ib → for m → for atom。
```

```cpp
for (auto&& type : atom.eachType()) {
    sf_cache[tid] = compute_sf(tau);           // 预计算所有原子的 sf

    for (auto&& bi : blocks)
        for (int ib = 0; ib < bi.nb; ++ib) {
            beta_q = interpolate(...);          // 1× per (l, ib)
            for (int m = -bi.l; m <= bi.l; ++m)
                for (auto& sc : sf_cache)
                    inner / update pass
        }
}
```

### 方案 C：当前循环次序 + 无缓存（现状）

```
完全保持原有结构。β 在原子循环内重复计算。
```

```cpp
for (auto&& type : atom.eachType()) {
    for (auto&& ad : atom.eachTypeAtom(type.ityp)) {
        sf_vals[ig] = sf(g, k);               // 1× per atom
        for (auto&& bi : blocks)
            for (int ib = 0; ib < bi.nb; ++ib) {
                beta_q = interpolate(...);      // n_atom 次重复
                for (int m = -bi.l; m <= bi.l; ++m)
                    inner / update pass
            }
    }
}
```

### 方案 D：atom 在最内层 + 无缓存

```
仅重排循环次序，不做任何缓存。sf_vals 在每个 (l, m) 组合内重复计算。
```

```cpp
for (auto&& type : atom.eachType()) {
    for (auto&& bi : blocks)
        for (int ib = 0; ib < bi.nb; ++ib) {
            beta_q = interpolate(...);          // 1× per (l, ib)
            for (int m = -bi.l; m <= bi.l; ++m)
                for (auto&& ad : atom.eachTypeAtom(type.ityp)) {
                    sf_vals[ig] = sf(g, k);     // n_proj 次重复
                    inner / update pass
                }
        }
}
```

## 计算成本对比

只计入**浪费**的开销（即 β/sf 的重复计算），inner/update passes 四种方案相同。

### 重复次数

| 方案 | β 重复倍率 | sf 重复倍率 |
|------|-----------|------------|
| A | n_β = 8 | n_atom = 100 |
| B | n_β = 8 | n_atom = 100 |
| C | n_atom × n_β = 800 | n_atom = 100 |
| D | n_β = 8 | n_atom × n_proj = 3200 |

### 耗时（n_atom = 100, n_q = 1000, 2 GHz）

| 方案 | β 开销 | sf 开销 | 总开销 | 占总 H|ψ⟩ 比 |
|------|--------|---------|--------|-----------|
| **A** | 8×1k×15 = 120k cyc | 100×1k×35 = 3.5M cyc | **3.6M cyc ≈ 1.8 ms** | **~10%** |
| **B** | 120k cyc | 3.5M cyc | **3.6M cyc ≈ 1.8 ms** | **~10%** |
| **C** | 800×1k×15 = 12M cyc | 3.5M cyc | **15.5M cyc ≈ 7.8 ms** | **~30%** |
| **D** | 120k cyc | 3200×1k×35 = 112M cyc | **112M cyc ≈ 56 ms** | **~75%** |

> 注：H|ψ⟩ 总时间包含动能项 + 局域势 FFT + 非定域项。对于 50³ FFT 网格 + 100 原子，~15–20 ms（计算限制，非带宽限制）。

### 随 n_atom 增长的趋势

```
A:  O(n_atom) —— sf 固定倍数，β 常数   →  3.5M + 常数
B:  O(n_atom) —— 同上                   →  3.5M + 常数
C:  O(n_atom) —— β 线性增长            →  3.5M + 0.12M × n_atom
D:  O(n_atom) —— sf 线性增长           →  1.12M × n_atom + 常数
```

## 内存成本对比

### 峰值内存

| 方案 | 新增持久缓存 | 临时 vector（按需分配） | 峰值总计 |
|------|------------|------------------------|---------|
| **A** | β 缓存: n_β × n_q × 8 = **64 KB** | sf_vals: n_q × 16 = 16 KB | **~80 KB** |
| **B** | sf 缓存: n_atom × n_q × 16 = **1.6 MB** | β 临时: n_q × 8 = 8 KB | **~1.6 MB** |
| **C** | 无 | sf_vals: 16 KB + β 临时: 8 KB × n_β × n_atom（逐次分配释放） | **~24 KB**（峰值，短期） |
| **D** | 无 | sf_vals: 16 KB × n_proj（逐次分配释放） | **~16 KB**（峰值，短期） |

> B 的 1.6 MB 是指单个 OpenMP 线程的。若有 N_th 个线程，总 sf 缓存达 N_th × 1.6 MB。

### 额外分配次数

重复的 `std::vector` 堆分配在方案 C/D 中不可忽略（每次 ~200 cycles + TLB 扰动）。

| 方案 | 堆分配次数（n_atom=100） | 释放次数 |
|------|-------------------------|---------|
| A | n_β = 8（β 缓存, 一次性）| 8 |
| B | n_atom = 100（sf 缓存, 一次性）| 100 |
| C | n_atom × (1 + n_β) = 900 | 900 |
| D | n_β × (1 + n_atom × n_proj / n_β) ≈ 3208 | 3208 |

### 缓存层次行为

| 方案 | 热数据驻留 | OpenMP 友好度 |
|------|-----------|--------------|
| **A** | β 缓存 64 KB 常驻 L2。sf_vals 16 KB 刚算完就进 inner/update 循环，L1 hit | ✅ β 缓存只读共享。sf_vals 各线程私有，无竞争 |
| **B** | sf 缓存 1.6 MB > 典型 L2 (512 KB)，退到 L3 或内存 | ⚠️ sf 缓存只读共享但体积大，线程数多时内存压力显著 |
| **C** | sf_vals 16 KB L1。β 每原子重算，刚算完就消费，L1 利用率高 | ⚠️ β 计算无数据竞争，但堆分配 900 次对 malloc 锁有压力 |
| **D** | sf_vals 16 KB 反复重算（3200 次），每次都新分配。TLB 和 L1 抖动 | ❌ sf 计算量大 + 分配频繁，OpenMP 加速比受分配器串行瓶颈限制 |

## 其他考量

### 代码结构

| 方案 | 循环嵌套 | 侵入性 |
|------|---------|--------|
| A | for atom → for bi → for ib → for m（不变） | 低：仅在 for type 层加一段缓存构建 |
| B | 改为 for bi → for ib → for m → for atom（重排） | 中：循环次序变化，HPSI_DEBUG 输出需适配 |
| C | 不变 | 零 |
| D | 重排 | 中：同 B |

### 与 HPSI_DEBUG 输出的兼容

- A：完全兼容。输出结构与现状一致。
- B：需要调整（atom 循环在最内层，per-atom 的 DBG 聚合需用索引累加器）。
- C：完全兼容。
- D：同 B。

## 结论

| 维度 | A（β 缓存） | B（sf 缓存） | C（现状） | D（重排无缓存） |
|------|-----------|------------|---------|--------------|
| 计算开销 | ✅ 最低 | ✅ 最低 | ⚠️ 中等 | ❌ 极高 |
| 峰值内存 | ✅ 64 KB | ⚠️ 1.6 MB | ✅ 0 | ✅ 0 |
| 分配次数 | ✅ 8 | ⚠️ 100 | ❌ 900 | ❌ 3208 |
| OpenMP | ✅ 只读共享 | ⚠️ 内存大 | ⚠️ 分配锁 | ❌ 分配锁+开销 |
| 代码侵入 | ✅ 低 | ⚠️ 中 | ✅ 无 | ⚠️ 中 |
| DBG 兼容 | ✅ 全兼容 | ⚠️ 需调整 | ✅ 全兼容 | ⚠️ 需调整 |

**推荐方案 A**：

- 与方案 B 计算成本完全相同，但**内存仅 64 KB**（B 的 1/25），天然 OpenMP 友好
- 循环结构不变，DBG 输出不变——改动最小
- 消除了 β 重复计算这个唯一真正的问题（对 100 原子省 ~6 ms/k-point）
- 不碰 sf_vals 和 Y_lm，这两个在原始结构中已经是最优位置

方案 C 在 n_atom 较小时（< 20）可接受，大体系下 β 重复成为瓶颈。方案 B 的 sf 缓存思路计算最优但内存代价高，仅适合内存充裕且 n_atom 不大的场景。方案 D 在任何场景下都不应使用。
