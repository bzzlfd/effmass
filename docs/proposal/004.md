# [proposal-004] SphericalBesselJ 小 x 递推精度限制

## 背景

`SphericalBesselJ` 在初始化时对 `j_0`、`j_1` 使用种子值（`x >= 0.5` 用 trig 直接公式，`x < 0.5` 用级数展开），随后通过三步递推计算更高 l：

```
j_{l+1}(x) = (2l+1)/x * j_l(x) - j_{l-1}(x)
```

当 x 较小时，`1/x` 因子放大种子值中的舍入误差，导致高 l 时数值崩溃。

## 精度数据

以直接 trig 公式 + 递推作为参考值，对比种子级数 + 递推的结果：

| x | l=0..1 | l=2 | l=3 | l=4 | l≥5 |
|---|--------|-----|-----|-----|-----|
| 0.3 | 精确 | 精确 | 精确 | ~1e-7 | 崩溃 |
| 0.1 | 精确 | 精确 | ~2e-8 | ~1e-4 | 崩溃 |
| 0.05 | 精确 | ~2e-10 | ~3e-6 | ~6e-2 | 崩溃 |
| 0.01 | 精确 | ~5e-7 | ~2e-1 | 崩溃 | 崩溃 |
| 0.001 | ~7e-11 | ~1e-3 | 崩溃 | 崩溃 | 崩溃 |

"崩溃"指相对误差 > 0.5。x ≥ 0.5 时对所有 l 精确（种子来自 trig 公式）。

## 原因

1. 种子 `j_0`、`j_1` 通过级数计算，精度 ~1e-16
2. 递推式中 `(2l+1)/x` 项将 ~1e-16 的误差放大为 `O(1e-16 * l/x)`
3. 当 `l/x ≳ 1e12` 时，误差达到 O(1)，结果完全不可用

经验安全区域：**`l ≤ floor(0.5 / x)`**

## 归档的替代方案

`namespace archived` 中的 `sphericalBesselJ(l, x)` 对 `x < 0.5` 使用**直接级数展开**计算任意 l，不受递推放大影响，在小 x 下保持 ~1e-16 精度。代价是每个 (l, x) 独立计算，无缓存复用。

## 可选的修复方向

1. **预扩展种子**：在 `initSeeds` 中，对 x < 0.5 预计算更多 l 的级数值（而非仅 j_0、j_1），缩短递推链长度
2. **混合策略**：按需在 `advance` 中根据 `l * x` 判断是否切换为直接级数计算
3. **保持现状**：主要使用场景（x = q*r 在赝势网格上）通常满足 `l ≪ 1/x`，不做修改

## 结论

暂不修改。当前 `SphericalBesselJ` 的主要场景是赝势傅里叶-贝塞尔变换中的 `q * r` 网格评估，x 值通常在 0.5 以上，递推稳定。对于极端小 x 高 l 的边缘情况，可使用 `archived` 中的直接 `sphericalBesselJ`。

## 后续：已采用修复方案

选择了混合策略：在 `advance()` 中按元素判断——`|x| >= SERIES_THRESH` 时用递推（高效），`|x| < SERIES_THRESH` 时调用模块级 `sphericalBesselJImpl` 直接级数展开（精确）。

修改位置：`src/math/sph_bessel.cppm:SphericalBesselJ::advance()` 第 195-199 行。

测试验证 `j_3(0.001) ≈ x³/105` 通过 `1e-18` 精度。
